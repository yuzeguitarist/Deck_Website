<p>搜索决定了一个剪贴板管理器好不好用。你一天可能复制几百条东西——如果找不到今早的那段代码，历史记录再多也是噪声。</p>
<p>云端应用可以把搜索丢给 Elasticsearch 或 Algolia。<a href="https://deckclip.app">Deck</a> 没有这个选项——它是一款 Swift / SwiftUI 构建的 macOS 原生剪贴板管理应用，所有数据都留在用户设备上。没有服务器，不发网络请求，没有遥测。搜索系统必须完全建立在 SQLite 和 Apple 原生框架之上。</p>
<p>本文介绍具体的实现方式。</p>
<hr>
<h2 id="挑战">挑战</h2>
<p>本地搜索的难度远超直觉。想想一个剪贴板管理器要处理什么：</p>
<p>standard tokenizers choke on <code>你好世界</code>.</p>
<ul>
<li><strong>CJK 文本</strong>没有空格分隔——标准分词器在 <code>你好世界</code> 面前束手无策。</li>
<li><strong>拼写错误和模糊召回</strong>——用户输入&quot;clpboard&quot;，仍然期望找到&quot;clipboard&quot;。</li>
<li><strong>语义意图</strong>——搜索&quot;网络错误&quot;应该能召回&quot;连接超时&quot;相关的条目。</li>
<li><strong>亚秒级延迟</strong>——搜索必须感觉是即时的，即使面对数万条记录。</li>
<li><strong>零资源浪费</strong>——常驻后台的应用不能浪费 CPU 和内存。</li>
</ul>
<p>Deck 的答案是一条<strong>五层搜索管线</strong>——每一层兜住上一层漏掉的结果。</p>
<hr>

<h2 id="第一层-fts5-全文检索-trigram-cjk-支持">第一层：FTS5 全文检索 + trigram CJK 支持</h2>
<p>which means Chinese, Japanese, and Korean text is treated as one giant token. Searching for <code>剪贴板</code> inside <code>这是一个剪贴板管理器</code> returns nothing.</p>
<p>SQLite 的 FTS5 是 Deck 搜索的主力。但 FTS5 的默认分词器按空白切分——中日韩文本会被当成一个巨大的 token。在 <code>这是一个剪贴板管理器</code> 中搜索 <code>剪贴板</code> 会一无所获。</p>
<p>解决方案：<strong>trigram 分词器</strong>。它不按词边界切分，而是用 3 字符滑窗切过文本，生成重叠的 token。<code>剪贴板管理</code> 变成 <code>剪贴板</code>、<code>贴板管</code>、<code>板管理</code>——子串搜索自然而然就行了。</p>

<h2 id="第二层-bitap-风格模糊匹配-fusesearch">第二层：Bitap 风格模糊匹配（FuseSearch）</h2>
<p>FTS5 快但死板——用户把&quot;screenshot&quot;打成&quot;screnshot&quot;时它帮不上忙。这就是 <code>FuseSearch</code> 登场的地方。</p>
<p>算法分两个阶段：</p>
<ol>
<li><strong>精确扫描</strong> — <code>O(n)</code> 子串搜索。命中则直接返回满分。</li>

<hr>
<h2 id="回顾">回顾</h2>
<p>没有哪种单一搜索技术能覆盖所有情况。FTS5 处理大多数查询很好用，但遇到拼写错误就不行，也做不了语义匹配。模糊搜索能容忍拼错，但在长文本上噪声很大。语义搜索能理解意图，但吃内存和 CPU。把它们堆在一起、让每层补另一层的短板，比试图造一个完美的搜索算法更现实。</p>
<p>CJK 支持是我们很早就做的决定。FTS5 的默认分词器按空白切分，中日韩文本基本上搜不了。trigram 分词解决了这个问题，代价是索引体积大约 3 倍——我们觉得这个取舍可以接受。</p>
<p>资源管理这块比预想的迭代了更多次。剪贴板管理器全天运行，嵌入模型又不小。我们经过了几轮调优才定下现在的模式：懒加载模型、最多缓存 400 个向量、空闲 30 秒后释放模型、矩阵乘法按 256 行分块执行以保持内存平稳。</p>
<p>本文描述的所有搜索能力都在用户的 Mac 上本地运行。没有数据离开设备，没有云端 API 调用。这是一开始就定下的约束，不是后来加的卖点。</p>

<h2 id="一个真实的例子">一个真实的例子</h2>
<p>你在终端里复制了一段 JSON 响应，想把它格式化成可读的样子。打开 Deck，输入一句话：</p>
<blockquote>
<p>&quot;帮我把最近复制的那段 JSON 格式化一下。&quot;</p>
</blockquote>
<p>Deck 的 AI Agent 随即完成了这些事：</p>
<ol>
<li>搜索剪贴板历史，找到 30 秒前复制的 JSON</li>
<li>发现没有现成的格式化插件，于是自己写了一个</li>
<li>将插件存入 <code>~/.deck/scripts/json-formatter/</code></li>
<li>执行插件，将 JSON 格式化为缩进整齐的格式</li>
<li>结果写入剪贴板历史，等你使用</li>
</ol>
<p>这不是一个对话框套了层皮。Deck 的 AI 有自己的工具集、权限边界、执行循环，还能通过创建插件来扩展自身能力。</p>

<h3 id="授权机制">授权机制</h3>
<p>注意表中 [Y] 标记的工具：所有写入、删除、创建类操作都需要用户明确授权。授权逻辑由 UI 层注入，缺失时默认拒绝。AI 能做很多事，但关键操作永远需要你点头。</p>
<p>另一个细节：工具结果存在截断机制（<code>maxToolSnippetCharacters = 2500</code>），防止超长输出耗尽 context window。</p>

<p>Deck 的 AI 不是一问一答的聊天机器人，而是一个循环执行的 Agent。System Prompt 中明确定义了行为模式：</p>
<div class="code-block-wrapper"><button class="copy-btn" aria-label="Copy code"><svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"><rect width="14" height="14" x="8" y="8" rx="2" ry="2"/><path d="M4 16c-1.1 0-2-.9-2-2V4c0-1.1.9-2 2-2h10c1.1 0 2 .9 2 2"/></svg></button><pre><code class="hljs">&lt;agent_loop&gt;

<h3 id="5-秒时限的实现">5 秒时限的实现</h3>
<p>JavaScriptCore 本身不提供公共的执行时限 API，Deck 通过私有 API 实现了这一功能：</p>
<div class="code-block-wrapper"><button class="copy-btn" aria-label="Copy code"><svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"><rect width="14" height="14" x="8" y="8" rx="2" ry="2"/><path d="M4 16c-1.1 0-2-.9-2-2V4c0-1.1.9-2 2-2h10c1.1 0 2 .9 2 2"/></svg></button><div class="code-lang-label">swift</div><pre><code class="hljs language-swift"><span class="hljs-keyword">static</span> <span class="hljs-keyword">let</span> scriptExecutionTimeout: <span class="hljs-type">TimeInterval</span> <span class="hljs-operator">=</span> <span class="hljs-number">5.0</span></code></pre></div><p>具体做法是通过 <code>dlopen</code> 加载 JavaScriptCore 框架，用 <code>dlsym</code> 获取 <code>JSGlobalContextGetGroup</code> 和 <code>JSContextGroupSetExecutionTimeLimit</code> 两个符号，封装为 <code>JSExecutionTimeLimiter</code> 类。在创建 JS 执行上下文时调用 <code>setLimit(group, timeLimit, callback, nil)</code>，确保任何脚本最多运行 5 秒。</p>
<p>需要网络权限的插件必须在 manifest 中声明 <code>requiresNetwork: true</code>，运行时使用独立的 ephemeral <code>URLSession</code>，不会泄露主应用的 cookies 或缓存。</p>

<h3 id="插件的积累效应">插件的积累效应</h3>

<h2 id="六-smart-rule-引擎-当条件遇上-ai">六、Smart Rule 引擎：当条件遇上 AI</h2>
<p>到目前为止都是&quot;用户主动触发&quot;的场景。Smart Rule 引擎让 AI 变成了一个后台自动化系统，<strong>在你复制的那一刻，规则就开始运转</strong>。</p>

<p>当规则触发 <code>aiPrompt(prompt)</code> 时，AI 自动介入处理：</p>
<ol>
<li><p>构造 <code>SmartRuleSourceSnapshot</code>，包含触发 item 的完整上下文（ID、类型、来源应用、内容、时间戳）</p>
</li>
<li><p>调用 <code>AIService.shared.runSmartRuleAutomation(prompt:source:)</code></p>
</li>
<li><p>在 <code>smartRule(source:)</code> 上下文中执行，自动授权（<code>autoApprove: true</code>）</p>
</li>
<li><p>工具作用域限定为当前触发的 item</p>
</li>
<li><p>AI 使用受限工具集（工具 1-7），不能在 Smart Rule 上下文中创建或修改插件</p>
</li>
</ol>
<h3 id="实际场景">实际场景</h3>
<ul>
<li><strong>Slack 链接提取 / Slack link extraction</strong>：来源是 Slack + 内容包含 URL → AI 自动提取链接摘要并打标签 / Source is Slack + content contains URL → AI extracts link summary and adds tags</li>
<li><strong>代码格式化 / Code formatting</strong>：内容类型是 code + 长度 &gt; 100 字符 → 自动用插件格式化 / Content type is code + length &gt; 100 chars → auto-format via plugin</li>
<li><strong>密码保护 / Password protection</strong>：来源是密码管理器 → 标记敏感 + 5 分钟后自动删除 / Source is a password manager → mark sensitive + auto-delete after 5 minutes</li>
</ul>

<h2 id="能力与约束-怎么划线">能力与约束：怎么划线</h2>
<p>Deck 的 Agent 设计始终在平衡一件事：给 AI 足够的能力去解决问题，同时确保它不会做出用户不想要的事。</p>
<p><strong>AI 能做什么：</strong></p>
<ul>
<li>12 个工具覆盖完整的剪贴板操作生命周期</li>
<li>自己编写和安装插件来扩展能力</li>
<li>流式 Agent 循环支持多步推理和工具链式调用</li>
<li>通过 Smart Rule 在后台自动运行</li>
</ul>
<p><strong>什么被限制了：</strong></p>
<ul>
<li>写入和删除操作需要用户授权</li>
<li>插件在严格的沙箱中执行，无文件系统、无 Shell、5 秒超时</li>
<li>Smart Rule 上下文中 AI 不能创建新插件，只能用现有工具</li>
<li>工具结果截断防止 context 爆炸</li>
<li>网络权限需要显式声明和用户确认</li>
</ul>
<p>这些限制不是因为不信任 AI。实际情况是：AI 可以自由创建插件，但需要你点确认；它可以在后台自动运行，但只有读取和搜索权限，没法自己装新插件。该放权的地方放权，该收紧的地方收紧。</p>
<p>这就是 Deck 的 AI Agent 系统：一套让 AI 真正参与产品工作流的架构，而不只是在旁边聊天。</p>

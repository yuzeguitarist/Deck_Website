隐私 政策

Deck 以本地优先隐私为基石——数据留在你的 Mac，网络访问最小化，敏感内容始终受到保护。

最后更新：2025年12月

数据存储

Deck 是一款本地优先应用。 剪贴板数据默认只存储在你的 Mac，不会默认上传到云端服务。

  • 剪贴板历史存储在本地 SQLite 数据库中
  • 图片和文件存储在本地应用数据目录中
  • 搜索索引在本地生成并存储
  • 设置与偏好保存在本地 UserDefaults 中

数据加密

Deck 使用现代加密能力保护敏感条目：

  • 受保护条目使用 AES-GCM 加密
  • 加密密钥存储在 macOS 钥匙串
  • 支持对指定剪贴板条目按需加密
  • 局域网共享使用端到端加密

生物识别安全

Deck 与 macOS 生物识别安全深度集成：

  • 可选的 Touch ID / Face ID 解锁
  • 访问敏感内容需要重新验证
  • 生物识别数据由 macOS 安全模块处理，Deck 无法访问

敏感内容保护

Deck 会在可识别场景下默认跳过敏感内容：

  • 识别密码管理器相关剪贴板类型
  • 检测包含敏感关键词的窗口标题
  • 支持手动将应用标记为「忽略」
  • 可配置的智能规则用于自动过滤

网络访问

Deck 的网络访问遵循“功能驱动、最小必要”原则：

  • 局域网共享: 仅在本地网络内进行 P2P 连接
  • 链接预览: 仅为预览读取页面元数据（可关闭）
  • iCloud 同步（可选）: 仅在你开启后，才会通过你的私有 iCloud 数据库同步加密数据
  • 上传分析数据（可选）: 默认关闭；开启后仅按设置项说明上传诊断数据
  • 无广告 SDK 或第三方追踪

数据删除

你可以随时删除数据：

  • 删除任何单个剪贴板条目
  • 清除所有历史记录
  • 卸载应用可能不会自动清除本地数据；如需彻底删除，请手动移除 ~/Library/Application Support/Deck 目录
  • 删除前可先导出备份

部分开源透明

Deck 采用部分开源模式。你可以：

  • 审计已发布模块来验证隐私行为
  • 了解数据如何在本地被处理和存储
  • 提交安全漏洞报告
  • 为开源模块贡献改进

源代码托管在 GitHub 上，采用 GPL-3.0 + Commons Clause 许可证。

如有隐私相关问题，请通过 GitHub Issues 联系我们。

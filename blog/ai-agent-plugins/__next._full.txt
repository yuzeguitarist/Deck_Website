<h2 id="a-real-example">A Real Example</h2>
<p>You copy a raw JSON response from a terminal and want it pretty-printed. Open Deck and type:</p>
<blockquote>
<p>&quot;Format the JSON I just copied.&quot;</p>
</blockquote>
<p>Deck&#39;s AI Agent then does the following:</p>
<ol>
<li>Searched clipboard history, located the JSON copied 30 seconds ago</li>
<li>No existing formatter plugin found, so it wrote one in JavaScript</li>
<li>Saved it to <code>~/.deck/scripts/json-formatter/</code></li>
<li>Ran the plugin, formatted the JSON with proper indentation</li>
<li>Result is ready in your clipboard history</li>
</ol>
<p>This isn&#39;t a chat UI bolted onto an LLM. Deck&#39;s AI has its own tool set, permission boundaries, execution loop, and the ability to extend itself by creating plugins.</p>
<hr>
<h2 id="multi-provider-abstraction-connecting-any-llm">Multi-Provider Abstraction: Connecting Any LLM</h2>
<p>Most AI apps lock you into a single vendor. Deck adopted a multi-provider architecture from day one. You can freely switch between OpenAI, Anthropic, Ollama local models, or even use the ChatGPT macOS client for free.</p>

<h3 id="authorization-mechanism">Authorization Mechanism</h3>
<p>Note the [Y] marks: all write, delete, and create operations require explicit user authorization. The auth logic is injected by the UI layer and defaults to rejection when absent. The AI can do a lot, but critical operations always need your approval.</p>
<p>Another detail: tool results are truncated at <code>maxToolSnippetCharacters = 2500</code> to prevent oversized outputs from exhausting the context window.</p>

<h2 id="agent-loop-think-act-observe">Agent Loop: Think, Act, Observe</h2>
<p>Deck&#39;s AI is not a simple Q&amp;A chatbot. It is a loop-driven Agent with an explicitly defined behavior pattern in the System Prompt:</p>
<div class="code-block-wrapper"><button class="copy-btn" aria-label="Copy code"><svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"><rect width="14" height="14" x="8" y="8" rx="2" ry="2"/><path d="M4 16c-1.1 0-2-.9-2-2V4c0-1.1.9-2 2-2h10c1.1 0 2 .9 2 2"/></svg></button><pre><code class="hljs">&lt;agent_loop&gt;
You are a capable agent. Act, don&#x27;t just describe.

For each request, follow this cycle:
1. Understand the user&#x27;s intent.
2. Plan internally...
3. Execute: call the necessary tool(s)...
4. Observe and verify each tool result before proceeding.
5. Respond to the user with the final result.

When issuing a tool call, output ONLY the tool call — no surrounding narration.
&lt;/agent_loop&gt;</code></pre></div><p><strong>Act, don&#39;t just describe</strong> sums up the whole approach. When you say &quot;find the links I recently copied,&quot; the AI won&#39;t respond with &quot;you can use the search feature to look for links.&quot; It calls <code>search_clipboard</code> directly, gets results, and tells you what it found.</p>

<p>The entire process is streamed via <code>AsyncThrowingStream&lt;AIStreamEvent, Error&gt;</code> with these event types:</p>
<p>Text fragment
Tool call request
Completion
Error</p>

<h3 id="how-the-5-second-timeout-works">How the 5-Second Timeout Works</h3>
<p>JavaScriptCore doesn&#39;t expose a public execution timeout API. Deck achieves this through private APIs:</p>
<div class="code-block-wrapper"><button class="copy-btn" aria-label="Copy code"><svg width="14" height="14" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round"><rect width="14" height="14" x="8" y="8" rx="2" ry="2"/><path d="M4 16c-1.1 0-2-.9-2-2V4c0-1.1.9-2 2-2h10c1.1 0 2 .9 2 2"/></svg></button><div class="code-lang-label">swift</div><pre><code class="hljs language-swift"><span class="hljs-keyword">static</span> <span class="hljs-keyword">let</span> scriptExecutionTimeout: <span class="hljs-type">TimeInterval</span> <span class="hljs-operator">=</span> <span class="hljs-number">5.0</span></code></pre></div><p>The approach uses <code>dlopen</code> to load the JavaScriptCore framework, <code>dlsym</code> to resolve <code>JSGlobalContextGetGroup</code> and <code>JSContextGroupSetExecutionTimeLimit</code>, wrapped in a <code>JSExecutionTimeLimiter</code> class. When creating a JS context, it calls <code>setLimit(group, timeLimit, callback, nil)</code>, ensuring no script can run longer than 5 seconds.</p>
<p>Plugins requiring network access must declare <code>requiresNetwork: true</code> in the manifest. Runtime uses an isolated ephemeral <code>URLSession</code>, so no cookies or caches from the main app are leaked.</p>

<h3 id="cumulative-plugin-effect">Cumulative Plugin Effect</h3>
<p>Back to the opening example:</p>
<p>User says: &quot;Format my JSON&quot;
AI searches history, finds the JSON
AI checks existing plugins, no JSON formatter
AI writes one, passes security checks, installs it
AI immediately uses the new plugin to format the JSON
Next time you need JSON formatted? The plugin is already there.</p>
<p>Every time the AI solves a new problem, it leaves behind a reusable plugin. As you use Deck over time, more plugins accumulate, and the system becomes increasingly capable.</p>

<p>   Construct a <code>SmartRuleSourceSnapshot</code> with full context of the triggering item (ID, type, source app, content, timestamp)</p>
<p>   Call <code>AIService.shared.runSmartRuleAutomation(prompt:source:)</code></p>

<p>   The AI uses a restricted tool set (Tools 1–7), with no plugin creation or modification allowed</p>

<h2 id="power-and-constraints-where-to-draw-the-line">Power and Constraints: Where to Draw the Line</h2>
<p>Deck&#39;s Agent design constantly balances one thing: giving the AI enough power to solve problems while ensuring it doesn&#39;t do anything the user didn&#39;t ask for.</p>
<p><strong>What the AI can do:</strong></p>
<p>12 tools covering the full clipboard lifecycle
Write and install plugins to extend its own capabilities
Streaming Agent loop for multi-step reasoning and chained tool calls
Run autonomously in the background via Smart Rules</p>
<p><strong>What&#39;s constrained:</strong></p>
<p>Write and delete operations require user authorization
Plugins run in a strict sandbox (no filesystem, no shell, 5s timeout)
In Smart Rule context, the AI cannot create new plugins, only use existing tools
Tool result truncation prevents context window exhaustion
Network permissions require explicit declaration and user confirmation</p>
<p>These constraints aren&#39;t about distrusting the AI. In practice, the AI can freely create plugins, but you have to confirm. It can run autonomously in the background, but only with read and search permissions; it can&#39;t install new plugins on its own. Power where it matters, limits where they&#39;re needed.</p>
<p>That&#39;s Deck&#39;s AI Agent system: an architecture where AI genuinely participates in the product workflow, rather than just chatting on the side.</p>
